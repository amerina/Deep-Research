# Deep Research（A 股公司深度调研版）

将用户提出的 A 股公司研究问题，转化为结构化、可追溯、可交付的研究结论。

默认入口是**单家公司深度研究**。只有用户明确要求比较两家或多家公司，或结论必须依赖横向参照时，才进入**公司对比分支**。

## 核心理念

- **先做公司研究，再做投资结论**
- **先钉牢公告与财报事实，再做推导**
- **优先解释“赚什么钱、怎么赚、为什么能持续”**
- **估值必须写明口径**：PE / TTM PE / PB / 股息率 / EV/EBITDA 等
- **中间文件默认可选**：仅在任务复杂、需要复核、需打包交付时再保存
- **结论必须区分三层**：已披露事实 / 管理层表述 / 研究推导

## 适用范围

### 默认适用

- 单家公司深度研究
- 公司质地与商业模式拆解
- 财务质量、成长性、分红能力、估值判断
- 周期、成长、价值、事件驱动的公司研究
- 投资决策前的快速深挖

### 分支适用

- 两家公司或多家公司横向对比
- 同行业龙头与跟随者比较
- 估值差异、经营质量差异、股东回报差异对比

### 不适用

- 宏观、行业、政策本身为唯一研究对象
- 纯技术产品、软件工具或通用知识调研
- 需要图谱而非报告的任务

## 研究入口判断

### 入口 A：单家公司深度研究（默认）

满足以下任一情况，直接走主流程：

1. 用户只提到一家公司
2. 用户问“这家公司值不值得看/买/跟踪”
3. 用户想判断股息、成长、利润质量、估值、风险
4. 用户没有明确给出对比对象

### 入口 B：公司对比研究（分支）

满足以下任一情况，进入对比分支：

1. 用户明确要求“甲和乙比”
2. 结论必须依赖可比公司参照
3. 研究目标是筛选行业内更优标的

**铁律**：没有明确对比需求时，不要强行切到对比模式。

## 资料分层（A 股版）

| 层级   | 资料类型                                                     | 可信度 | 用法                           |
| ------ | ------------------------------------------------------------ | ------ | ------------------------------ |
| **L1** | 年报、半年报、季报、招股书、募集说明书、权益分派公告、业绩预告、交易所问询与回复、监管处罚公告 | ✅ 高   | 核心事实必须优先来自这里       |
| **L2** | 公司官网、投资者关系活动记录、业绩说明会纪要、官方公众号、券商原始引用的公司发言 | ✅ 较高 | 用于补足经营细节与管理层表述   |
| **L3** | 主流财经媒体、专业数据库、卖方研报摘要、行情聚合页           | ⚠️ 中   | 用于辅助，不作唯一支撑         |
| **L4** | 社区讨论、股吧、论坛、自媒体                                 | ❓ 低   | 仅用于发现争议点，不用于下结论 |

### A 股必查资料清单

单家公司研究至少优先覆盖其中大部分：

1. 最近 3-5 年年报
2. 最近一期半年报或季报
3. 招股书或最近一次重大融资文件
4. 最近 12 个月的重要临时公告
5. 权益分派/分红公告
6. 投资者关系活动记录
7. 交易所问询函与回复（如有）
8. 股权激励、回购、减持、关联交易、并购重组公告（如有）

## 事实标注规则

所有关键陈述必须明确属于以下三层之一：

| 层级           | 含义                                     | 标注格式       | 示例                                      |
| -------------- | ---------------------------------------- | -------------- | ----------------------------------------- |
| **已披露事实** | 财报、公告、监管披露中已确认的信息       | `[已披露]`     | `[已披露] 2024 年归母净利润 5.16 亿元`    |
| **管理层表述** | 业绩会、调研纪要、 IR 活动中的管理层说法 | `[管理层表述]` | `[管理层表述] 新产能预计明年释放`         |
| **研究推导**   | 基于事实与对比得到的判断                 | `[研究推导]`   | `[研究推导] 当前估值已部分反映高分红预期` |

**铁律**：

- 不得把管理层表述写成已披露事实
- 不得把研究推导写成公司已确认结论
- 不得把二手媒体口径当作一手公告口径

## 工作方式

### 默认模式：轻量模式

默认不强制保存中间文件。可以直接在对话中完成：

1. 问题拆解
2. 资料收集
3. 事实提取
4. 推导结论
5. 最终输出

### 升级模式：留痕模式

满足以下任一条件时，建议保存中间文件：

1. 用户明确要求完整底稿
2. 研究问题复杂，涉及多轮迭代
3. 需要独立 Agent 校验
4. 需要打包交付或后续复用

### 推荐目录（可选）

仅在留痕模式下使用：

├── 05.5_校验记录.md

## 主流程：单家公司深度研究

**默认走这条流程。**

### Step 0：任务识别与边界界定

先明确：

1. 研究对象是谁
2. 研究时间点是什么
3. 用户要判断的是价值、成长、周期、事件，还是综合研究
4. 最终要回答的是“是什么”“好不好”“贵不贵”“风险在哪”

推荐拆成 3-5 个子问题：

1. 公司靠什么赚钱，业务结构是否清晰
2. 公司的竞争位置与护城河如何
3. 财务质量、成长性、现金流、分红能力如何
4. 当前估值与市场预期是否匹配
5. 关键风险、催化剂、跟踪指标是什么

### Step 1：锁定资料源与时间窗口

高优先级顺序：

1. 最近 3-5 年年报
2. 最近一期半年报/季报
3. 最新权益分派、业绩预告、重大事项公告
4. 招股书与再融资文件
5. 投资者关系活动记录、问询函与回复
6. 辅助行情与媒体资料

**敏感信息必须写日期**：

- 股价
- 市值
- PE / PB / 股息率
- 产能、订单、项目进度
- 股东增减持、回购、股权激励

### Step 2：建立单公司研究框架

单家公司默认从以下 8 个维度组织研究：

1. **商业模式**：卖什么、客户是谁、收入与利润来自哪里
2. **行业位置**：行业空间、竞争格局、公司份额与定位
3. **成长性**：收入、归母净利润、扣非净利润、ROE/ROIC 的趋势
4. **财务质量**：毛利率、净利率、现金流、资产负债、应收/存货/资本开支
5. **股东回报**：分红比例、股息率、回购、资本配置
6. **估值**：PE、PB、股息率、EV/EBITDA 等，必须写明口径与日期
7. **风险**：需求、价格、政策、治理、商誉、减值、关联交易、单一客户依赖
8. **催化剂与跟踪指标**：新产品、产能释放、提价、行业景气、政策变化

### Step 3：提取事实卡片

事实卡片优先记录以下类型：

1. 近 3-5 年收入、归母、扣非、经营现金流
2. 关键业务收入与毛利结构
3. 主要客户/供应商集中度
4. 分红方案、股利支付率、股息率口径
5. 估值指标及其对应日期
6. 重大资本运作：回购、定增、可转债、并购、减持、股权激励
7. 风险信号：存货异常、应收恶化、减值、诉讼、处罚、治理争议

如果需要留痕，可使用 `templates/fact-card.md` 作为底稿模板；如果不留痕，可以直接在对话中编号引用。

### Step 4：形成研究主线

根据公司类型选择主线，不要平均用力：

- **价值/红利型**：分红、现金流、资产负债表、估值安全边际
- **成长型**：收入增速、利润弹性、产能与产品周期、行业渗透率
- **周期型**：价格、库存、开工率、成本曲线、供需拐点
- **事件驱动型**：并购重组、资产注入、激励、政策变化、诉讼处罚

### Step 5：从事实到结论

每个结论都必须按以下结构表达：

1. `[已披露]` 事实是什么
2. `[管理层表述]` 补充了什么
3. `[研究推导]` 因此可以得到什么判断
4. 结论的置信度高/中/低

禁止出现以下偷换：

- 用“利润增长”替代“扣非利润增长”
- 用“拟分红”替代“已实施分红”
- 用“静态 PE”替代“TTM PE”而不说明
- 用“管理层计划”替代“业绩已兑现”

### Step 6：估值判断

估值必须明确回答：

1. 用什么估值指标
2. 指标对应什么日期
3. 为什么这个指标适合这家公司
4. 当前估值对什么预期做了定价

示例：

- 红利股：股息率 + PE + 分红可持续性
- 银行/保险：PB + ROE
- 周期股：PB / EV/EBITDA / 周期中枢利润
- 成长股：PE 或 PEG，但要解释增速是否可持续

### Step 6.5：独立 Agent 校验（复杂任务建议，重要任务强烈建议）

当研究将用于正式判断、公开输出或金额较大的投资决策时，启动独立 Agent 校验以下内容：

1. 关键数字是否准确
2. 估值口径是否一致
3. 是否把拟披露、管理层表述、研究推导混为一谈
4. 是否遗漏了核心风险或用户最关心的指标

如果任务只是一次轻量问答，可以跳过此步。

### Step 7：场景验证

用一个最典型的投资问题做 sanity check：

- 如果这是高股息逻辑，分红下修会怎样
- 如果这是成长逻辑，增速放缓后估值还能否成立
- 如果这是周期逻辑，价格回落后的利润中枢是多少
- 如果这是事件逻辑，若事件落空，回到什么基本面估值

### Step 8：输出最终结论

最终输出至少回答以下问题：

1. 这家公司最核心的看点是什么
2. 当前最关键的风险是什么
3. 估值是便宜、合理还是偏贵
4. 哪些指标值得继续跟踪

## 分支流程：公司对比研究

只有在明确需要比较公司时才进入。

### 对比对象选择原则

优先选择：

1. 同行业、同商业模式公司
2. 同产业链位置公司
3. 同估值框架公司

避免无效对比：

- 把红利股和高成长股硬放在同一结论框架里
- 把强周期股和稳定消费股只按 PE 比
- 忽略股本结构、业务结构、并表口径差异

### 对比维度

默认从以下维度比较：

1. 商业模式与利润来源
2. 行业地位与竞争格局
3. 收入与利润成长性
4. 财务质量与现金流
5. 分红与股东回报
6. 估值水平与市场预期
7. 风险暴露与治理结构
8. 适合哪类投资风格

### 对比分支铁律

- 先分别研究公司 A 和 B，再做比较
- 不得跳过单公司研究，直接抄表格对比
- 必须说明可比性边界，不可比的地方要明确写出

## 质量检查清单

### 核心检查

- [ ] 研究对象与时间点已明确
- [ ] 单公司研究已优先完成；若是对比，已先分别研究各公司
- [ ] 核心结论主要由 L1/L2 资料支撑
- [ ] 已区分已披露事实、管理层表述、研究推导
- [ ] 估值口径和日期已明确写出
- [ ] 关键风险没有遗漏
- [ ] 结论与公司类型匹配，没有把价值、成长、周期、事件逻辑混淆

### A 股专项检查

- [ ] 最近 3-5 年财务趋势已覆盖
- [ ] 扣非净利润与归母净利润关系已检查
- [ ] 经营现金流与利润匹配情况已检查
- [ ] 分红、回购、融资、减持等股东回报与资本配置已检查
- [ ] 重要公告、问询函、处罚、关联交易等治理风险已检查
- [ ] 股价、市值、PE、PB、股息率等市场数据已标注日期

### 对比分支检查

- [ ] 对比对象具备可比性
- [ ] 对比口径一致
- [ ] 不可比的地方已单独说明
- [ ] 结论不是“谁都好”，而是明确谁更适合什么风格

## 最终回复规范

### 默认输出

最终回复应包含：

1. **一句话核心结论**
2. **关键发现摘要**：3-5 点
3. **核心风险**
4. **估值/股东回报口径说明**：必要时简写

### 单家公司研究推荐结构

```markdown
# [公司名] 研究结论

## 一句话结论

## 关键发现
1. ...
2. ...
3. ...

## 关键风险

## 估值与跟踪指标
```

### 公司对比研究推荐结构

```markdown
# [公司A] vs [公司B]

## 一句话结论

## 胜负手维度

## 关键差异

## 谁更适合哪类风格
```

### 不默认要求

以下内容只有在用户明确需要时才提供：

- 中间文件清单
- 工作目录结构
- `outcome.tar.gz` 打包路径
- 全量底稿文件
